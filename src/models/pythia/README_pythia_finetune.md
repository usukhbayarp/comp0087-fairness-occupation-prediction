# pythia_finetune.py

This script fine-tunes a Pythia language model for 20-class occupation classification using LoRA (Low-Rank Adaptation). 

## Dependencies

Install all required packages before running the script:

```bash
!pip install transformers==4.40.0 peft==0.10.0 accelerate bitsandbytes scikit-learn tqdm
```

Each line in both files follow this schema:

```json
{"id": 0, "text": "...", "label": "professor", "gender": "M"}
```

## Input

`train.jsonl` and `dev.jsonl` generated by `export_dataset_jsonl.py` in Part 1 are needed.

## Supported Model Sizes

| `--model_size` | Parameters |
|----------------|------------|
| `70m` | 70 M |
| `160m` | 160 M |
| `410m` | 410 M |
| `1b` | 1 B |
| `1.4b` | 1.4 B |
| `2.8b` | 2.8 B |

## How to Run

### Basic Usage (410m, no quantisation)

```bash
!python pythia_finetune.py \
    --model_size 410m \
    --data_dir ./processed \
    --output_dir ./checkpoints
```

### QLoRA — Large Model (1.4b)

```bash
!python pythia_finetune.py \
    --model_size 1.4b \
    --data_dir ./processed \
    --output_dir ./checkpoints \
    --use_4bit \
    --train_batch_size 64
```

## CLI Arguments

| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| `--model_size` | str | `410m` | Pythia variant: `70m` \| `160m` \| `410m` \| `1b` \| `1.4b` \| `2.8b` |
| `--data_dir` | str | *(required)* | Folder containing `train.jsonl` and `dev.jsonl` |
| `--output_dir` | str | `./checkpoints` | Root directory for all saved checkpoints |
| `--num_epochs` | int | `3` | Maximum number of training epochs |
| `--train_batch_size` | int | `128` | Per-device training batch size |
| `--eval_batch_size` | int | `128` | Per-device evaluation batch size |
| `--grad_accum` | int | `1` | Gradient accumulation steps (increase if OOM) |
| `--lr` | float | `2e-4` | Peak learning rate (cosine schedule with warmup) |
| `--max_length` | int | `256` | Maximum token length; longer texts are truncated |
| `--max_train_samples` | int | `None` | Cap number of training samples (for quick testing) |
| `--use_4bit` | flag | `False` | Enable QLoRA 4-bit quantisation (recommended for 1b+) |
| `--seed` | int | `42` | Random seed for reproducibility |

## Output

After training completes, all outputs are written under `--output_dir`:

```
<output_dir>/
└── pythia-<model_size>/              ← one folder per model size
    ├── checkpoint-<step>/            ← intermediate checkpoints (one per epoch)
    │   ├── adapter_model.safetensors
    │   ├── adapter_config.json
    │   └── trainer_state.json
    └── best/                         ← BEST checkpoint (selected by macro F1)
        ├── adapter_model.safetensors ← LoRA weights only (small file)
        ├── adapter_config.json       ← LoRA architecture config
        ├── tokenizer.json            ← tokenizer vocabulary
        ├── tokenizer_config.json
        └── label_meta.json           ← label mappings (required by eval script)
```

## Training Behaviour

The Trainer monitors macro F1 on the dev set at the end of every epoch. The checkpoint with the highest macro F1 is automatically restored at the end of training and saved to the `best/` directory

### LoRA Configuration

| Parameter | Value | Meaning |
|-----------|-------|---------|
| `r` (rank) | 16 | Rank of the low-rank decomposition — higher = more capacity |
| `lora_alpha` | 32 | Scaling factor; effective learning rate scales as `alpha / r` |
| `lora_dropout` | 0.1 | Dropout applied to LoRA layers for regularisation |
| `target_modules` | `query_key_value` | Targets Pythia's merged QKV projection in every attention layer |
| `bias` | `none` | Bias terms are not trained |
